# 규칙 — 진실의 원천(SoT)·우선순위·충돌 해소

> 사실은 오직 `.claude/spec/` 에서만. 이 파일은 **어느 문서가 무엇의 권위인지**와 **충돌 시 누가 이기는지**만 정한다. spec 내용을 여기 복붙하지 않는다.

## 1. 권위 도메인 (문서별 관할)
| 문서 | 최종 권위를 갖는 영역 |
|---|---|
| `03_..._api_spec.md` | **API 계약** — 엔드포인트·필드·메서드·상태코드·에러 envelope·enum·토큰 정책 |
| `01_...wireframe_spec.md` | **화면 동작/IA/상태전이/검증 규칙/도메인 데이터 사전**, 그리고 §9 = **디자인 토큰** |
| `04_...frontend_spec.md` | **프론트 아키텍처·기술 스택·Step 절차·횡단정책(인증/에러/폼/이탈보호)**, 그리고 §2 **문서 정합성 노트 = 충돌 해소 결정** |
| `DS-01` | **디자인 토큰·컴포넌트 명명·상태 변형·아이콘**(§1~7) |
| `DS-02` | **16개 화면별 시각 명세**(레이아웃·필드·모드) |
| `DS-00` / `DS-03` / `DS-04` | 개요·상호작용 패턴·프롬프트 템플릿 |

## 2. 충돌 우선순위 (위가 이긴다)
1. **명시적 결정** — `04 §2 문서 정합성 노트` 및 `.claude/rules/decisions.md`(D1~). (예: 주관식 정답 단일/콤마 모델이 DS-02/03의 "다중 정답 N개" 묘사를 **무시**한다.)
2. **해당 영역의 관할 문서**(위 표).
3. 그 외 문서.
4. **코드 ↔ spec 충돌** → 항상 spec 우선. 코드를 spec 에 맞춘다.

## 3. 멈춤 규칙 (환각 금지)
- spec 에 **없거나 서로 모순**이고 §2/decisions 로도 해소 안 되면 → **코드 작성 금지**, 사용자에게 질문(🙋🏻).
- 명세에 없는 엔드포인트/필드/화면을 추측으로 만들지 않는다. → [[antipatterns]]
- **Figma 는 spec 문서가 아니라 시각(레이아웃/구성) 참조다.** 값은 DS-01 토큰 경유(raw 값 복사·새 토큰 추가 금지), **동작/데이터/계약은 무권한**(01/03/04). 시각 정책·우선순위 = [[ui-conventions]].
- Figma 에 요소가 "없음"은 충돌이 아니라 gap → 명세/DS/토큰으로 시각 보완(**중단·질문 없이**). 단 **동작/데이터/계약 gap 은 보완 금지, 멈춤(🙋🏻)**.

## 4. spec 는 읽기전용
- `.claude/spec/` 의 어떤 파일도 수정/생성/덮어쓰기 금지. PreToolUse 훅이 차단한다. → [[hooks]]
- `DS-01`(토큰·컴포넌트)·`DS-02`(화면 시각 명세)는 **채워져 있고 권위**다. 특정 시각 디테일이 명세에 없거나 서로 모순일 때만 추측 말고 멈춰 질문(🙋🏻).
