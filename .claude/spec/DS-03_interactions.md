# Gravit 백오피스 — 상호작용 명세 (Claude Design 입력용)
 
> **이 문서의 역할**: 시각 디자인 1차 완료 후, 모달·토스트·검증·상태 전이 같은 **상호작용**을 추가하기 위한 문서입니다. 인터랙티브 프로토타입 단계에서 사용하세요.
>
> **사용 방법**: 시각 디자인이 안정되면 이 문서의 해당 섹션을 발췌하여 Claude Design에 추가 입력하세요.

## 1. 모달 패턴
 
### 1-1. Confirm Modal (기본)
 
| 사양 | 값 |
|---|---|
| 폭 | 400px |
| 컴포넌트 | shadcn/ui Dialog |
| 닫기 트리거 | ESC, 외부 클릭, 우상단 X 버튼 |
 
**구조**:
```
┌──────────────────────────────────────┐
│  타이틀                           ✕  │
├──────────────────────────────────────┤
│  설명 텍스트                          │
├──────────────────────────────────────┤
│              [취소]  [확인]          │
└──────────────────────────────────────┘
```
 
- 타이틀: 16px bold
- 설명: 14px regular
- 액션 버튼: 우측 정렬, 8px 간격
- [취소]: **Outline Button**
- [확인]: variant는 액션 위험도에 따라 결정
### 1-2. Confirm Modal Variant 매핑
 
| 액션 | [확인] Variant | 색상 |
|---|---|---|
| 유저 status → SUSPENDED | Primary | Primary |
| 유저 status → ACTIVE | Primary | Primary |
| **유저 status → DELETED** | **Destructive** | 빨강 |
| 유저 role → USER | Primary | Primary |
| **유저 role → ADMIN** (권한 상승) | **Destructive** | 빨강 |
| 신고 isResolved 토글 | Primary | Primary |
| **공지 DELETE** | **Destructive** | 빨강 |
 
### 1-3. Strict Match Modal (스테이징 promote 전용)
 
> **가장 위험한 액션**의 안전장치. 라벨명을 정확히 입력해야만 [반영] 버튼이 활성화됩니다.
 
| 사양 | 값 |
|---|---|
| 폭 | **480px** (입력 영역 확보) |
| 컴포넌트 | shadcn/ui Dialog 기반 (커스텀) |
 
**구조**:
```
┌──────────────────────────────────────────────────┐
│  반영 완료 처리하시겠습니까?                  ✕  │
├──────────────────────────────────────────────────┤
│  ⚠️ 이 작업은 prod에 INSERT를 실행하며           │
│     되돌릴 수 없습니다.                          │
│                                                  │
│  반영할 라벨:                                    │
│  ┌────────────────────────────────────────────┐ │
│  │ 2026-04-25-update                          │ │ ← 읽기 전용, monospace
│  └────────────────────────────────────────────┘ │
│                                                  │
│  진행하려면 아래에 라벨명을 정확히 입력하세요:   │
│  ┌────────────────────────────────────────────┐ │
│  │                                            │ │ ← 입력 input, monospace
│  └────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────┤
│                          [취소]  [반영]          │
└──────────────────────────────────────────────────┘
```
 
- 경고 박스: 배경 `danger-bg`, 텍스트 `danger-text`, ⚠️ 아이콘
- 라벨명 표시 영역: 코드 블록 또는 read-only input, **monospace**
- 입력 input: monospace, placeholder 없음
- **[반영] 활성 조건**: 입력값 === 라벨명 (대소문자/공백 포함 strict 비교)
- [반영]: **Destructive Button**
- ESC / 외부 클릭 시: 모달 닫힘 + 입력값 초기화

## 3. 폼 검증 패턴
 
### 3-1. 검증 실패 시각 표시
 
```
┌──────────────────────────────────────────────┐
│ 제목 *                                        │
│ ┌──────────────────────────────────────────┐ │
│ │                                          │ │ ← 빨간 보더
│ └──────────────────────────────────────────┘ │
│ ⚠ 제목은 필수 항목입니다.                     │ ← 12px, danger-text
└──────────────────────────────────────────────┘
```
 
- 입력 필드 보더: `danger-text` 컬러
- 에러 메시지: input 하단, 12px, `danger-text`
- 검증 통과 시 보더와 메시지 즉시 제거
### 3-2. 검증 트리거 시점
 
| 트리거 | 동작 |
|---|---|
| 입력 중 (onChange) | 검증 안 함 (피드백 과다 방지) |
| 포커스 아웃 (onBlur) | 단일 필드 검증 |
| [저장] / [게시] 등 제출 클릭 시 | **전체 필드 검증** + 실패 시 토스트 "필수 항목을 확인해주세요." |
 
### 3-3. 화면별 검증 규칙
 
#### NOTICE_NEW / NOTICE_DETAIL
 
| 필드 | 검증 |
|---|---|
| 제목 | 빈 값 불가 |
| 요약 | 빈 값 불가 |
| 본문 | 빈 값 불가 |
 
#### CHAPTER_DETAIL / UNIT_DETAIL / LESSON_DETAIL (편집 모드)
 
| 필드 | 검증 |
|---|---|
| 제목 | 빈 값 불가 |
| 설명 (해당 시) | 빈 값 허용 |
 
#### PROBLEM_DETAIL — 객관식 편집
 
| 필드 | 검증 |
|---|---|
| 지시문 | 빈 값 불가 |
| 본문 | 빈 값 불가 |
| 옵션 4개의 보기 본문 | 모두 빈 값 불가 |
| 옵션 4개의 해설 | 모두 빈 값 불가 |
| 정답 (radio) | 정확히 1개 선택 |
 
#### PROBLEM_DETAIL — 주관식 편집
 
| 필드 | 검증 |
|---|---|
| 지시문 | 빈 값 불가 |
| 본문 | 빈 값 불가 |
| 정답들의 본문 | 모두 빈 값 불가 |
| 정답들의 해설 | 모두 빈 값 불가 |
| 정답 개수 | 최소 1개 |
 
#### STAGING_DETAIL — 모든 폼
 
객관식/주관식 검증 규칙은 PROBLEM_DETAIL과 동일.
 
#### 라벨명 입력 모달
 
| 필드 | 검증 |
|---|---|
| 라벨명 입력 | 라벨명과 정확히 일치 (strict match) |

## 4. 상태 표시 패턴 (Empty / Loading / Error)
 
### 4-1. Empty State (데이터가 없을 때)
 
```
                     📭 (아이콘 48px, 회색)
                  데이터가 없습니다
              조건에 맞는 항목이 없습니다.
              [+ 새로 만들기 (해당 시)]
```
 
| 상황 | 메시지 |
|---|---|
| 검색·필터 적용 후 결과 없음 | "조건에 맞는 항목이 없습니다." |
| 기본 (데이터 자체가 없음) | "{도메인}이 없습니다." (예: "공지가 없습니다.") |
 
- 아이콘: 회색, 48px
- 메시지: 16px, `text-secondary`, 중앙 정렬
- 보조 액션 버튼: 해당하는 경우만 (예: 공지 목록의 [+ 공지 작성])
### 4-2. Loading State
 
- **Skeleton**: 테이블 행 모양의 회색 placeholder bar
- 5~10줄 표시 후 실제 데이터로 교체
- 전체 페이지 로딩이 아닌, 데이터 영역만 skeleton 처리
### 4-3. Error State (데이터 로딩 실패)
 
```
                     ⚠️ (아이콘 48px, 회색)
              데이터를 불러올 수 없습니다
              잠시 후 다시 시도해주세요.
                   [다시 시도]
```
 
- 아이콘: 회색, 48px
- 메시지: 16px, `text-secondary`, 중앙 정렬
- [다시 시도]: **Outline Button**

## 5. 화면별 상호작용
 
### 5-1. LOGIN
 
| 트리거 | 동작 |
|---|---|
| OAuth 버튼 클릭 | 해당 provider의 OAuth 팝업 또는 리다이렉트 |
| 로그인 성공 (role=ADMIN) | 대시보드(`/`)로 이동 |
| 로그인 실패 또는 role=USER | 카드 하단에 에러 메시지: "백오피스 접근 권한이 없습니다." |
 
### 5-2. DASHBOARD
 
| 트리거 | 동작 |
|---|---|
| Stat Card 1 클릭 | `/users` 이동 |
| Stat Card 2 클릭 | `/staging/labels` 이동 |
| Stat Card 3 클릭 | `/reports` 이동 |
| 페이지 진입 | 카운트 1회 호출 (자동 갱신 없음) |
 
### 5-3. NOTICE_LIST
 
| 트리거 | 동작 |
|---|---|
| 행 클릭 | `/notices/:id` 이동 |
| [수정] 클릭 | `/notices/:id` 이동 |
| [+ 공지 작성] 클릭 | `/notices/new` 이동 |
| 페이지 변경 | 해당 페이지 데이터 호출 |
 
### 5-4. NOTICE_NEW
 
| 트리거 | 동작 |
|---|---|
| [취소] 클릭 (변경 내용 있음) | confirm 모달: "변경 사항이 있습니다. 나가시겠습니까?" |
| [취소] 클릭 (변경 내용 없음) | 즉시 목록으로 이동 |
| [임시저장] 클릭 | 검증 → status=DRAFT로 저장 → 목록으로 이동 + Success 토스트 |
| [게시] 클릭 | 검증 → status=PUBLISHED로 저장 → 목록으로 이동 + Success 토스트 ("공지가 게시되었습니다.") |
 
### 5-5. NOTICE_DETAIL
 
#### 표시 모드
 
| 트리거 | 동작 |
|---|---|
| [편집] 클릭 | 편집 모드로 전환 |
| [삭제] 클릭 | Confirm Modal (Destructive): "공지를 삭제하시겠습니까?" → 확인 시 삭제 + 목록 이동 + Success 토스트 |
 
#### 편집 모드
 
| 트리거 | 동작 |
|---|---|
| 상태 드롭다운 옵션 | 현재 상태에 따라 제한:<br>- DRAFT → DRAFT, PUBLISHED<br>- PUBLISHED → PUBLISHED, ARCHIVED<br>- ARCHIVED → ARCHIVED만 (드롭다운 disabled) |
| [취소] 클릭 (변경 내용 있음) | confirm 모달 |
| [저장] 클릭 | 검증 → 저장 → 표시 모드로 전환 + Success 토스트 |
 
### 5-6. USER_LIST
 
| 트리거 | 동작 |
|---|---|
| 행 클릭 | `/users/:id` 이동 |
| 검색 input + Enter / 검색 버튼 클릭 | 검색 실행 |
| 필터 드롭다운 변경 | **즉시 적용** (별도 [적용] 버튼 없음) |
 
### 5-7. USER_DETAIL
 
| 트리거 | 동작 |
|---|---|
| 역할 드롭다운에서 다른 값 선택 | Confirm Modal 표시:<br>- → ADMIN: **Destructive** "관리자 권한을 부여하시겠습니까?"<br>- → USER: Primary "관리자 권한을 회수하시겠습니까?" |
| 상태 드롭다운에서 다른 값 선택 | Confirm Modal 표시:<br>- → DELETED: **Destructive** "유저를 삭제 처리하시겠습니까?"<br>- 그 외: Primary "유저 상태를 변경하시겠습니까?" |
| Confirm Modal [확인] | API 호출 → 성공 시 UI 갱신 + Success 토스트 / 실패 시 Error 토스트 + 드롭다운 원복 |
| Confirm Modal [취소] | 드롭다운 원복 |
 
### 5-8. REPORT_LIST

### 5-9. REPORT_DETAIL

| 처리상태 드롭다운 "해결됨 → 미해결" | Confirm Modal → 성공 시 **상세 페이지에 머무름** (의도 외 동작 방지) |

### 5-11. PROBLEM_DETAIL

|---|---|
| [편집] 클릭 | 편집 모드로 전환 |

| [취소] 클릭 (변경 내용 있음) | confirm 모달 |
| [저장] 클릭 | 검증 → 저장 → 표시 모드 + Success 토스트 |
 
### 5-12. STAGING_LIST
 
| 트리거 | 동작 |
|---|---|
| 행 클릭 | `/staging/labels/:label` 이동 |
| 필터 변경 | 즉시 적용 |
| **첫 진입 시 기본 필터** | 상태 = "PENDING" |
 
### 5-13. STAGING_DETAIL ⭐ 가장 복잡
 
#### 좌측 리스트 항목 전환
 
| 트리거 | 동작 |
|---|---|
| 다른 항목 클릭 (미저장 변경 있음) | **자유 이동** (모달 없음). 입력값은 메모리 유지, 좌측 ● 점 유지 |
| 다른 항목 클릭 (미저장 변경 없음) | 자유 이동 |
| 동일 항목으로 다시 돌아옴 | 입력값 복원 |
| 페이지 이탈 (브라우저 뒤로/새로고침/다른 라우트) | beforeunload confirm 또는 라우터 confirm |
 
#### 우측 폼 — 변경 감지
 
| 트리거 | 동작 |
|---|---|
| 입력값 변경 (초기값과 다름) | input 좌측 4px Primary 보더 표시 + 좌측 리스트의 ● 점 표시 + [저장] 버튼 활성화 |
| 입력값을 원래 값으로 되돌림 | 보더 제거 + ● 점 제거 + [저장] 비활성 |
 
#### 우측 폼 — 저장
 
| 트리거 | 동작 |
|---|---|
| [저장] 클릭 | 검증 → 변경된 필드만 다중 PATCH (Promise.allSettled) |
| 전체 성공 | baseline 갱신 + 보더/● 모두 제거 + Success 토스트 "저장되었습니다." |
| 부분 실패 | 성공한 필드만 baseline 갱신, 실패 필드는 입력값/보더/● 유지 + Error 토스트 "{N}건 저장 실패." |
| 전체 실패 | 모든 필드 유지 + Error 토스트 "저장에 실패했습니다." |
| 저장 진행 중 | [저장] 버튼 비활성 + spinner, 폼 input 비활성 |
 
#### 헤더 — [반영 완료 처리] 버튼
 
| 상태 | 버튼 |
|---|---|
| status === COMPLETED | **숨김** |
| status === PENDING & 미저장 항목(●) 존재 | **비활성** + hover 툴팁 "저장하지 않은 항목이 있습니다 ({N}건)" |
| status === PENDING & 미저장 항목 없음 | **활성** (Destructive variant) |
 
#### 헤더 — 반영 흐름
 
| 트리거 | 동작 |
|---|---|
| [반영 완료 처리] 클릭 | **Strict Match Modal** 오픈 |
| 모달에서 라벨명 입력 (정확히 일치) | [반영] 버튼 활성화 |
| 모달 [반영] 클릭 | API 호출 |
| 성공 | Success 토스트 "라벨이 반영되었습니다." → **목록(`/staging/labels`)으로 자동 이동** (필터 유지) |
| 실패 | 모달 닫힘 + Error 토스트, 상세 페이지 머무름 |
| ESC / 외부 클릭 | 모달 닫힘 + 입력값 초기화 |
 
#### COMPLETED 상태 진입 시
 
| 영역 | 동작 |
|---|---|
| 안내 배너 | 페이지 상단 노출 (Banner Success) |
| 헤더 [반영 완료 처리] | 숨김 |
| 좌측 리스트 | 정상 동작, 단 ● 점 표시 안 함 |
| 우측 input/textarea/radio | Read-only 상태 |
| [저장] 버튼 | 숨김 |

## 6. 글로벌 상호작용
 
### 6-1. 라우팅 / 네비게이션
 
| 트리거 | 동작 |
|---|---|
| 사이드바 메뉴 클릭 | 해당 경로 이동 + 활성 상태 표시 |

### 6-2. 페이지네이션
 
| 트리거 | 동작 |

| 첫 페이지에서 [이전] | [이전] 버튼 disabled |
| 마지막 페이지에서 [다음] | [다음] 버튼 disabled |

| 우상단 X | 모달 닫힘 |
