# Gravit 백오피스 — 프로젝트 개요 (Claude Design 입력용)
 
> **이 문서의 역할**: Claude Design과의 첫 대화에서 프로젝트 컨텍스트를 설정하기 위한 문서입니다. 함께 첨부할 문서: `DS-01_design_system.md` (디자인 시스템).
 
---
 
## 1. 프로젝트 한 줄 정의
 
**Gravit 백오피스 (Admin Dashboard)** — CS(컴퓨터 과학) 학습 서비스의 내부 운영자용 관리 도구.
 
## 2. 사용자 / 사용 환경
 
| 항목 | 내용 |
|---|---|
| **사용자** | 내부 운영자 (수십 명 이내) |
| **사용 환경** | 사무실 데스크탑, 업무 시간 중 사용 |
| **기술 친숙도** | 높음 (내부 직원, 학습 곡선 허용 가능) |
| **사용 빈도** | 매일 (특히 스테이징 검수 작업이 빈번) |
 
## 3. 디바이스 / 뷰포트 — **데스크탑 전용**
 
> ⚠️ **중요**: 이 프로젝트는 **데스크탑 전용**이며 **반응형 디자인이 필요하지 않습니다**.
 
| 항목 | 값 |
|---|---|
| **타겟 디바이스** | 데스크탑 PC만 |
| **최소 뷰포트** | 1280px |
| **권장 뷰포트** | 1280px ~ 1920px |
| **모바일 / 태블릿** | **미지원** (디자인 변형 불필요) |
| **반응형 break point** | **없음** (단일 데스크탑 레이아웃만) |
 
→ Claude Design에서 mobile/tablet 변형을 생성하지 마세요. 데스크탑 단일 디자인만 필요합니다.
 
## 4. 디자인 톤 & 무드
 
| 항목 | 값 |
|---|---|
| **전체 톤** | 미니멀 & 정보 중심 (information-dense) |
| **레퍼런스** | Linear, Vercel Dashboard 계열 |
| **컬러 모드** | **라이트 모드 전용** (다크 모드 불필요) |
| **분위기** | 차분하고 전문적, 장식적 요소 최소화 |
| **정보 밀도** | 높음 — 운영자가 한 화면에서 많은 정보를 빠르게 파악해야 함 |
 
### 디자인 원칙
 
1. **기능 우선**: 화려함보다 정보 가독성과 작업 효율
2. **일관성**: 동일한 패턴은 동일한 형태로 (모달, 버튼, 테이블 등)
3. **여백 절제**: 정보 중심이지만 호흡 가능한 여백 유지
4. **포인트 컬러 절제 사용**: Primary 컬러(`#7738EE`)는 액션과 강조에만, 큰 면적 사용 금지
5. **위험 시각화**: 위험한 액션은 명확한 시각 차이 (빨강 destructive)
## 5. UI 라이브러리 / 기술 스택
 
| 항목 | 값 |
|---|---|
| **UI 라이브러리** | **shadcn/ui** (Radix UI + Tailwind CSS) |
| **아이콘** | **lucide-react** |
| **폰트** | 시스템 폰트 + **Pretendard** (한글) |
| **언어** | 한국어 |
 
### shadcn/ui 사용 원칙
 
- 모든 UI 컴포넌트는 shadcn/ui 기본 컴포넌트를 사용합니다.
- 커스텀 컴포넌트는 shadcn/ui를 확장하는 형태로만 만듭니다.
- shadcn/ui의 기본 디자인 토큰(border-radius 6px 등)을 따릅니다.
## 6. 화면 구성 (총 16개 화면)
 
```
🔐 /login                            로그인 (사이드바 없음, 독립 레이아웃)
📊 /                                 대시보드 (위젯 3개)
👤 /users                            유저 목록
   /users/:userId                    유저 상세
🚨 /reports                          신고 목록
   /reports/:reportId                신고 상세
📚 /chapters                         챕터 목록
   /chapters/:chapterId              챕터 상세
   /units/:unitId                    유닛 상세
   /lessons/:lessonId                레슨 상세
   /problems/:problemId              문제 상세
🧪 /staging/labels                   스테이징 라벨 목록
   /staging/labels/:label            스테이징 라벨 상세 (가장 복잡)
📢 /notices                          공지 목록
   /notices/new                      공지 작성
   /notices/:noticeId                공지 상세/수정
```
 
## 7. 글로벌 레이아웃
 
```
┌─────────────────────────────────────────────────────────────────────────┐
│┌────────────────┐┌──────────────────────────────────────────────────────┐│
││ Sidebar 240px  ││ Header 56px                                          ││
││                ││ ┌──────────────────────────────┬──────────────────┐  ││
││ Gravit Admin   ││ │ Breadcrumb / Page Title      │   [로그아웃]     │  ││
││                ││ └──────────────────────────────┴──────────────────┘  ││
││────────────────││──────────────────────────────────────────────────────││
││ 📊 대시보드    ││                                                      ││
││ 👤 유저 관리   ││                                                      ││
││ 🚨 신고 관리   ││  Content Area                                        ││
││ 📚 학습 컨텐츠 ││  - max-width 1200px                                  ││
││ 🧪 스테이징    ││  - 좌우 32px padding, 상단 24px padding              ││
││ 📢 공지 관리   ││                                                      ││
││────────────────││                                                      ││
││ 👤 운영자 이름 ││                                                      ││
│└────────────────┘└──────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────────────┘
전체 min-width: 1280px (= 240px 사이드바 + 1040px 컨텐츠 영역)
```
 
| 영역 | 폭 / 높이 | 사양 요약 |
|---|---|---|
| Sidebar | 240px | 흰 배경, 우측 1px border, 메뉴 활성 시 좌측 3px Primary 보더 |
| Header | 높이 56px | 흰 배경, 하단 1px border, breadcrumb 또는 페이지 타이틀 |
| Content | max-width 1200px | 좌우 32px / 상단 24px padding |
| 페이지 배경 | - | `#FAFAFA` (옅은 회색) |
 
> 로그인 화면(`/login`)은 사이드바 없이 독립 레이아웃입니다.
